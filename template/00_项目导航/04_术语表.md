# 术语表

| 术语 | 含义 |
| --- | --- |
| PO | Product Owner，产品负责人 |
| SM | Scrum Master，流程守护者 |
| TL | Tech Lead，技术负责人 |
| FS | Full-stack，全栈工程师 |
| DoR | Definition of Ready，就绪定义 |
| DoD | Definition of Done，完成定义 |
| AC | Acceptance Criteria，验收标准 |
| ADR | Architecture Decision Record，架构决策记录 |
| Sprint Goal | 当前 Sprint 要实现的单一目标，见当前 Sprint 的 `00_Sprint计划.md` |
| Story | 描述用户/业务价值的 Backlog 条目，不等于开发 Task |
| AC | Story 的可验证验收条件 |
| Task | 为实现 Story 而拆出的可执行工作，唯一运行事实源是当前 Sprint 任务表 |
| 模块 | 功能、页面、服务、数据库或基础设施边界；是拆分概念，不要求对应固定目录或文档 |
| Sprint Backlog | 当前 Sprint 选入的 Story 和实现工作 |
| Sprint 任务表 | `03_迭代运行/<当前Sprint>/01_Sprint任务表与流程看板.md` §4 |
| 输入输出总表 | `00_项目导航/06_团队输入输出总表/`，只登记跨 Sprint 长期交付事实 |
| 证据 | 证明 Task/AC 完成的代码提交、PR、CI、测试、截图或结论链接 |
| 当前 Sprint | 由 `00_项目导航/00_项目首页.md` 显式指向，不按目录最大编号猜测 |
| Increment | 满足 DoD 的可工作增量 |
| worktree | Git 多工作区机制 |
