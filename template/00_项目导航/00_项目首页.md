## 30 分钟上手

> **本页是团队唯一总入口**，新成员从这里开始。

1. 读本页的项目背景、Sprint Goal 和仓库策略。
2. 打开 `11_角色行动手册.md` 找到**你的角色卡**：职责/边界/输入输出/必读最小集/周期职责。
3. 打开 `../03_迭代运行/Sprint-0-启动/01_Sprint任务表与流程看板.md` §4，找到自己的 Task、前置和证据。
4. 按角色卡“必读最小集”读规范（不必通读全部）；与 Reviewer 走读 AC。
5. **跑通第一个 S 任务并过 DoD**（这是“已上手”的证据，不是写心得）。
6. **首签由项目创建者准备、SM 跟踪、成员本人执行。** 若创建时已自动发布首签
   Notice，你只需运行其中给你的完整 `sign` 命令（含 `--notice=<sha256>`）；
   若首页尚无 Notice，创建者按 `09_SM教练查询与回复模板.md` §10.1 完成 Git
   事实源和 `bootstrap`，SM 不需要重新设计一份通知。
   工具从角色事实源取得本次提交身份并用互斥锁串行写入，**不要为签核修改仓库
   `user.name/user.email`**；禁止编辑他人 Event 或旧台账。逾期仍可签（advisory），工具记录 late。

不要求通读知识库。遇到问题再按主题查阅。
