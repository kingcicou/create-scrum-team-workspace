## 最小启动顺序

1. 全员从 `00_项目导航/00_项目首页.md` 的“30 分钟上手”开始。
2. PO/TL/资深成员确认 Goal、Story/AC、模块拆分和任务复杂度。
3. 执行成员在 Sprint 任务表找到自己的 Task，只更新状态和证据。
4. 执行仓库动作：{{REPO_ACTION}}。
5. Sprint 结束前分别确认 Goal、遗留和下一轮准入。
