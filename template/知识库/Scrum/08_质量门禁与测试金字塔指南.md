# 质量门禁与测试金字塔指南

> 质量内建。不是末期检查，是每个角色在每个阶段的承诺。

## 1. 质量左移三阶段

| 阶段 | 质量动作 | 主责 |
| --- | --- | --- |
| Refinement | AC 含失败/边界/异常；识别测试点；契约先行 | PO + QA + TL |
| 开发 | 单测/集成测试与功能代码同步；CI 绿；自测通过再 PR | 编码角色 |
| 验收/发布 | E2E 与回归；缺陷分级；Go/No-Go 结论 | QA + FS |

## 2. 测试金字塔（默认建议）

```text
        E2E（少而精，覆盖关键用户路径）
       ────────────────────────────────
      集成 / 契约（API、组件交互）
     ──────────────────────────────────
    单元 + 快照（每模块/每组件，高频运行）
```

| 层级 | 数量级 | 速度 | 主责 |
| --- | --- | --- | --- |
| 单元 | 多 | 毫秒 | 编码角色（BE/FE） |
| 集成/契约 | 中 | 秒 | Mid.BE（API 侧）+ Mid.FE（前端联调） |
| E2E | 少 | 分钟 | Mid.FE + FS（CI 中调度） |

反金字塔（顶部胖 E2E）会让 CI 又慢又脆。SM 应在 Retro 监测层级比例。

## 3. 三道质量门禁

| 门禁 | 检查项 | 不通过的处理 |
| --- | --- | --- |
| DoR 门禁 | AC、依赖、估算、风险、设计就绪 | Story 退回 Backlog Refinement |
| PR 门禁 | 代码评审、单测/集成通过、CI 绿、自测证据 | PR 不合并 |
| DoD 门禁 | AC 全满足、测试证据、PO 验收、FS 集成结论 | Story 不标 Done，不进 Review |

门禁结论与产物评审分开：方案approved不代表测试或CI门禁passed。关闭与waiver规则见`14_Sprint关闭与证据治理规范.md`。

## 4. CI 最小要求

| 阶段 | 内容 |
| --- | --- |
| 提交触发 | 静态检查、单测、构建产物 |
| PR 合并前 | 集成测试、契约测试、覆盖率阈值 |
| 集成分支 | 关键 E2E、性能回归基线、安全扫描 |
| 预发布 | 完整 E2E + 手动验收 |
| 发布 | 制品签名/版本号、回滚预案就绪 |

> 真实 CI 实现写入 `06_发布运维/02_CI_CD/`，本指南只定底线。

## 5. 缺陷分级

| 级别 | 定义 | 处理时限 |
| --- | --- | --- |
| P0 阻断 | 主流程不可用、数据风险、安全漏洞 | 当日处理，停发布 |
| P1 严重 | 关键场景受损，无 workaround | 本 Sprint 修复 |
| P2 一般 | 次要功能或体验受损 | 进入 Backlog 排期 |
| P3 改进 | 优化、轻度体验问题 | 视容量排入后续 Sprint |

缺陷台账放 `05_质量验证/03_缺陷分析/`；修复 PR 必须关联缺陷 ID。

## 6. 性能与安全基线

| 维度 | 最小基线 | 落地位置 |
| --- | --- | --- |
| 性能 | 关键接口 p95、首屏 LCP、关键交互 INP | `05_质量验证/04_性能与安全/` |
| 安全 | 依赖漏洞扫描、SAST、密钥泄露检查 | CI + `05_质量验证/04_性能与安全/` |
| 可访问性 | WCAG 2.1 AA 关键控件 | `04_工程设计/04_前端设计系统/04_可访问性/` |

任何降级要进 ADR，不能默默放行。

## 7. Done 证据链

任一 Story 的 Done 必须能回答以下问题：

- AC 列表逐条结果？
- 单测/集成/E2E 报告链接？
- PR 与 CI 链接？
- 缺陷状态？
- PO 验收结论？
- FS 集成与回滚说明？

“计划在下个Sprint执行”不是当前通过证据。命令结果至少记录命令、时间、退出码和摘要或报告链接。
测试文件数、测试函数数或手工汇总数字只能描述规模，不能单独证明“全部通过”或
“状态稳定”。质量结论必须关联实际命令、commit、执行结果和失败明细；统计口径
变化时保留原值并解释差异，不事后改写。

模板已经在 `02_产品待办/03_AC_验收标准/` 与 Story 工作区预留位置。

## 8. 质量反模式

| 反模式 | 后果 | 修正 |
| --- | --- | --- |
| QA 末期集中测试 | 缺陷雪崩 | 质量左移到 Refinement |
| 覆盖率当唯一指标 | 假安全感 | 与缺陷率、E2E 通过率联合看 |
| 全量手工回归 | 慢且不可靠 | 关键路径自动化 |
| 红灯过夜 | 团队对 CI 失去信任 | 红灯即停车，FS/提交人立刻处理 |
| 性能/安全靠"上线再说" | 难以补救 | 基线进 Sprint 0 与 DoD |
| 用测试文件数/函数数推断稳定 | 未执行或失败用例被统计掩盖 | 绑定实际命令、commit、结果和失败明细 |
