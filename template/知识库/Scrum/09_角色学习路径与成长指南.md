# 角色学习路径与成长指南

> 每个角色在团队中既要"会用",也要"会传"。这份指南提供入职阅读路径、技能进阶台阶和成长产物。

## 1. 通用入职 30 分钟阅读路径

任何角色第一天先读：

1. `00_项目导航/00_项目首页.md`
2. `00_项目导航/02_角色与联系方式.md`
3. `00_项目导航/05_输入输出管理规范.md`
4. `知识库/Scrum/00_Scrum团队协同知识总纲.md`
5. `知识库/Scrum/02_角色协同与能力模型.md`

读完应能回答：本团队的事实源在哪？我应该把产物放哪？谁会消费它？

## 3. 能力进阶台阶（通用）

| 阶段 | 自我标准 | 团队信号 |
| --- | --- | --- |
| L1 上手 | 能按规范产出，知道找谁 | 总表填写正确，无误投目录 |
| L2 独立 | 能拆分任务、写 AC、做 Review | PR 一次过率提升 |
| L3 影响 | 能识别反模式、修复机制、带新人 | 改进项台账中有他主推的条目 |
| L4 传承 | 能反向回流模板和知识库 | 知识库有他署名的演进 |

晋阶不是头衔，而是行为证据。建议每 2-3 个 Sprint 在 `07_度量改进/04_能力矩阵.md` 复核一次。

## 4. 个人成长产物

每位成员都应保留：

- **学习心得**（`知识库/Scrum/学习心得_<Name>.md`）：阶段性总结，至少每 1-2 个 Sprint 更新一次。
- **反人格清单**：写下自己最容易滑入的反人格场景与对应触发器。
- **改进承诺**：每次 Retro 自己认领的改进项。

学习心得是个人产物，但内容可被同岗位新人复用，是团队的人才资产。

## 5. 跨角色补能

| 想补什么 | 推荐路径 |
| --- | --- |
| 想理解 PO 视角 | 旁听 1 次 Refinement + 读 `07_产品发现与价值排序指南.md` |
| 想理解 SM 工作 | 跟着 SM 做一次站会 + 读 `03_落地执行与SM作战指南.md` |
| 想理解架构思路 | 跟随 TL 走读一次 ADR 评审 |
| 想补 QA 视角 | 跟 Mid.BE/Mid.FE 一起做一次 AC 拆解 |
| 想补 DevOps | 跟 FS 走一遍发布与回滚 |

互相补能能扩大巴士因子，也减少"隐形工作"被某个人独自扛。

## 6. 成长反模式

| 反模式 | 后果 | 修正 |
| --- | --- | --- |
| 只学方法论不出产物 | 知识不沉淀 | 必须有可被引用的输出 |
| 学习心得当周报 | 知识库变垃圾场 | 心得偏总结/取舍/反人格 |
| 把帽子当头衔 | 团队僵化 | 帽子是协作接口，可轮换 |
| 不留接班材料 | 巴士因子=1 | 主动写"我的怪癖与上下文"段落 |
| 不复盘自己 | 反复掉同一个坑 | Retro 单独认领一条自我改进 |
