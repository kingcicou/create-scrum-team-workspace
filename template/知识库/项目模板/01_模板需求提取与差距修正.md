# 模板需求提取与差距修正

## 1. 设计结论

模板必须优先复用团队实践中证明有效的机制，而不是只参考工程脚手架目录。

必须保留：

- `05_输入输出管理规范`
- `06_团队输入输出总表`
- `SM_作战手册`
- `TeamWork/worktree` 工程协同
- 项目类型化 Sprint 0
- 文档主事实源边界
- 知识库沉淀机制

## 2. 与单人工程脚手架的区别

| 单人工程脚手架 | Scrum 团队协同模板 |
| --- | --- |
| 关注代码目录 | 关注价值流和协作闭环 |
| README 说明为主 | 05/06/SM/总表驱动执行 |
| docs 可在仓库内 | 文档主事实源在项目工作区 |
| 一人直接开发 | 多角色 worktree/分支/FS 集成 |
| 少量模板文件 | Sprint、质量、发布、度量、知识库齐备 |
