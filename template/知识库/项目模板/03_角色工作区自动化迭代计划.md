# 角色工作区自动化迭代计划

> 版本说明：本计划记录 v0.3 的角色 worktree 自动化设计。自 v0.6 起，
> “默认创建目标仓”仅适用于 `create/import/rewrite`；`reuse` 只登记并接入现仓。

**迭代主题：** 角色确认后统一生成编码工作区  
**决策日期：** 2026-06-29  
**适用版本：** 0.3.0

## 1. 目标

把角色命名、邮箱、工作目录、分支、Git 身份、远端初始化从分散说明升级为可审阅、可执行、可验证的生成流程。

## 2. 需求与采纳结论

| 建议 | 结论 | 落地 |
| --- | --- | --- |
| 先列出角色，再调整名称和邮箱 | 采纳 | 交互模式先展示全表，可按槽位或全部编辑 |
| 确认后统一创建角色工作目录 | 采纳 | 为 5 个编码角色创建 worktree |
| 自动设置每个工作区 Git 身份 | 采纳并修正实现 | 使用 `extensions.worktreeConfig` + `--worktree` |
| 提供远端地址后初始化提交 | 部分采纳 | 本地初始化提交默认创建；远端只在显式确认后 push |
| 每个角色提交开发测试 commit | 作为可选能力采纳 | 只在个人分支生成就绪文件，不自动合入 |
| FS 维护主代码目录 | 采纳 | 主目录配置 FS 身份，承担集成；FS 编码仍使用独立 worktree |
| 保留 workspace Git 默认模式 | 不采纳为默认 | 默认改为独立代码仓 `repo`，减少文档仓与代码仓耦合 |

## 3. 实施项

| ID | 工作项 | 验收 |
| --- | --- | --- |
| P0-1 | 角色配置预览、按槽位编辑、最终确认 | 生成前可看到名称、邮箱、目录 |
| P0-2 | 邮箱格式与角色唯一性校验 | 重名、重复邮箱、非法邮箱会中止 |
| P0-3 | 初始化 `main`、`sprint-n` 和 5 个 worktree | 分支和目录数量正确 |
| P0-4 | worktree 级身份隔离 | 5 个 worktree 可读到各自身份 |
| P1-1 | 可选角色就绪测试提交 | commit 作者与角色一致 |
| P1-2 | 可选 origin 配置与显式 push | 本地裸远端验证全部分支 |
| P1-3 | 配置文件可复现新字段 | Git、Sprint、远端、测试提交均可配置 |
| P1-4 | 操作指南和稳定知识回流 | SOP、Scrum 知识库、README 同步 |

## 4. 风险控制

| 风险 | 控制 |
| --- | --- |
| worktree 身份互相覆盖 | 禁止普通 local config，使用 worktree config |
| 分支引用冲突 | 个人分支使用 `feature/sprint-n/...` |
| 空远端、错误远端或已有历史 | `--remote` 与 `--push` 分离；push 不加 force |
| 占位邮箱进入远端历史 | `--push` 前强制检查所有角色真实邮箱 |
| 测试提交污染主线 | 不自动合并角色分支 |
| 自动化无法代替权限管理 | 仓库创建、授权和凭据仍由平台/FS 管理 |

## 5. 验证场景

1. 默认创建：独立代码仓、Sprint 分支、5 个角色 worktree。
2. 自定义角色：名称和邮箱同步进入配置、目录与 Git 身份。
3. 身份提交：5 个测试 commit 的作者各自正确。
4. 本地裸远端：显式 push 后存在 `main`、`sprint-n` 和 5 个角色分支。
5. 关闭能力：`--no-worktrees`、`--no-git` 不创建角色工作区。
6. dry-run：只展示计划，不写文件、不访问远端。
