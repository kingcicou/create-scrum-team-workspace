# Sprint {{SPRINT_NUMBER}} 质量门禁清单

> **本清单在 Sprint Planning 时由 TL / 领域负责人与 SM 共同填写。**
> Review 时逐条核对，未通过的门禁必须记录原因和处置（waiver / 延后 / 阻塞）。
> 不新增文件：本清单作为 Sprint 工件的一部分，与 Sprint 任务表配合使用。

## 使用说明

- **何时填**：Sprint Planning 期间，选定 Story 后、开工前。
- **谁填**：TL / 领域负责人提候选门禁，SM 确认可判定性。
- **何时查**：Daily 快速扫阻塞项；Review 逐条核对。
- **硬规则**：门禁标准必须客观可判定，不写"基本满足""差不多"。

## 状态词典

| 状态 | 含义 |
| --- | --- |
| 🔵 待填 | Planning 时尚未确认 |
| 🟢 通过 | 客观标准已满足，有证据 |
| 🟡 waiver | 本轮豁免，记录原因和下一轮补回计划 |
| 🔴 阻塞 | 未满足，阻断相关 Story/Sprint 关闭 |
