## 如何创建（技术清晰后执行）

代码仓由独立工具 `tools/setup-code-repo.mjs` 落地，四种策略：

| 策略 | 行为 | 命令 |
|---|---|---|
| **create** | 新建独立代码仓、初始化分支、可选角色 worktree | `node tools/setup-code-repo.mjs --strategy=create --repo={{REPO_NAME}} --sprint=0 [--worktrees] [--remote=<url>] [--push]` |
| **reuse** | 登记现有仓库；不复制、不改写历史 | `node tools/setup-code-repo.mjs --strategy=reuse --repo={{REPO_NAME}} --source=<url\|path>` |
| **import** | 创建目标仓与导入清单；不自动迁移历史 | `node tools/setup-code-repo.mjs --strategy=import --repo={{REPO_NAME}} --source=<url\|path>` |
| **rewrite** | 创建候选替代仓；登记旧仓、切换门禁与回退路径 | `node tools/setup-code-repo.mjs --strategy=rewrite --repo={{REPO_NAME}} --source=<旧仓>` |

**安全边界**：代码仓位于文档仓内部（`10_代码仓库/<repo>/`）。工具会**先把该路径写入
文档仓 `.gitignore`，再执行嵌套 `git init`**，保证文档仓与代码仓 Git 历史相互独立、
文档仓不误跟踪代码。`import`/`rewrite` 的危险历史迁移**必须人工评审后执行**，工具不自动进行。

执行后本卡状态由工具更新为 `decided`，并在 `10_代码仓库/00_仓库清单.md` 登记一行。
