## 如何创建（技术清楚后，经结构化审批）

仓库决策是**技术与投资决定**（PO/TL 审批），与"人员规范签核"是**不同工件**，
不共用同一 Campaign。采用 `propose → approve → check → apply`，而非一命令直接建仓：

```bash
# 1) 生成决策提案与 dry-run，不建仓（策略：create/reuse/import/rewrite）
node tools/setup-code-repo.mjs propose --strategy=create --repo={{REPO_NAME}} \
  --tech=<技术栈> --rationale=<ADR/Spike依据> [--remote=<url,不含凭据>]

# 2) PO 与 TL 分别审批（留痕于 Git）
node tools/setup-code-repo.mjs approve --decision=REPO-001 --actor=po
node tools/setup-code-repo.mjs approve --decision=REPO-001 --actor=tl

# 3) 复核创建条件
node tools/setup-code-repo.mjs check --decision=REPO-001

# 4) 审批通过后执行（保留一次交互确认；--yes 只跳过输入，不绕过审批门禁）
node tools/setup-code-repo.mjs apply --decision=REPO-001
```

**apply 门禁**：决策状态=approved、PO+TL 已审批、文档仓干净、目标目录空、父文档仓已
忽略目标代码目录、远端不含凭据、重复执行不再建第二套。**危险历史迁移（import/rewrite）
必须人工评审后执行**，工具不自动进行。

> 决策工件位于 `.team/repo-decisions/REPO-NNN.json`，与 `.team/signoffs/` 的人员签核
> **分属不同工件**——"签了规范"不等于"批准这次技术投资"。
