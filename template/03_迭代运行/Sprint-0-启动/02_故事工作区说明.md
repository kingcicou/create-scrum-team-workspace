# 故事工作区说明

每条进入 Sprint 的 Story，建议建立一份工作区文档，记录需求、分支、PR、测试证据和验收结论。

## 文件命名

```text
Story-<ID>_<short-title>.md
```

示例：

```text
Story-ARK-101_login-baseline.md
```
