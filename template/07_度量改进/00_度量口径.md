# 度量口径

度量用于改进，不用于个人考核。

| 指标 | 说明 | 数据来源 |
| --- | --- | --- |
| Sprint Goal 达成 | 本 Sprint 目标是否达成 | Review |
| 完成率 | Done Story / 计划 Story | 看板 |
| 阻塞数 | 阻塞登记数量 | 风险与障碍 |
| 缺陷数 | Sprint 内发现缺陷 | 缺陷分析 |
| CI 稳定性 | 构建失败次数 | CI |

## 治理健康度（文档与事实源）

把“事实源失真”“文档杂乱”这类坑变成可持续监控，而非靠一次评审才发现。

| 指标 | 口径 | 采集 | 目标 |
| --- | --- | --- | --- |
| 事实源一致率 | 总表/任务表声明状态 == 文档 `status` 的抽样比例 | 每周抽样对照 | ≥ 95% |
| approved 证据覆盖率 | approved 产物中附可验证证据的比例 | 抽查证据链接 | ≥ 90% |
| 元数据健康度 | 仅 managed 文档的缺失/非法字段率、未标注率、孤儿率 | 索引脚本自动算 | 纳管文档无治理债 |
| 文档精简度 | 同主题重复文件数 / 总数 | 索引对照 | 趋近 0 |

采集节奏：Sprint 末或异常触发时跑 `python tools/generate_doc_index.py`，读“治理健康度”与
“缺字段/停滞审计”；每 Sprint 末抽样核对一致率与证据覆盖率，记入度量快照。
